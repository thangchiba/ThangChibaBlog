### **プロジェクトの背景**

- **ユーザーのニーズ:** 私の妹は、いくつかのアパートを所有しており、使いやすいアプリケーションを必要としていました。彼女はコンピュータースキルが限られており、Excelのような複雑なツールを使うことができなかったので、シンプルでアクセスしやすいソリューションが必要でした。
- **アクセシビリティ要件:** PCやスマートフォンなど、さまざまなデバイスからいつでもアクセスできるプラットフォームである必要がありました。
- **プラットフォームの選択:** マルチプラットフォームに対応するため、Webベースのアプリケーションを選択しました。

### **サーバーレスを選んだ理由**

- **低頻度利用に最適化:** アプリケーションは月に数回しかアクセスされないものの、常に利用可能な状態である必要があるため、サーバーレスインフラストラクチャが最適な選択肢として浮上しました。このアプローチにより、リソースが効率的に活用され、アプリケーションの使用パターンを反映することができます。
- **AWSサーバーレスインフラストラクチャの活用:** AWSサービスを利用する決定は、費用対効果が高く、効率的でスケーラブルなソリューションが必要だったためです。
  - **強固な認証:** AWS Cognitoは、安全なユーザー管理機能を提供し、ユーザーデータの安全とプライバシーを確保するために採用されています。
  - **効率的なフロントエンドホスティング:** AWS Route53、CloudFront、S3バケットの組み合わせにより、信頼性が高くスケーラブルなホスティングソリューションを提供します。CloudFrontのCDN（コンテンツデリバリーネットワーク）との統合により、世界中どこからでも高速にアクセスできます。
  - **柔軟なバックエンドサービス:** AWS Lambda関数とAPI Gatewayがバックエンドのバックボーンを形成し、応答性が高く適応性のある環境を提供します。ここでの主な利点は、24時間365日アクセスできる必要があるサービスにとって重要な、常に利用可能であることです。
  - **データベースソリューション:** DynamoDBは、AWSサーバーレスアーキテクチャとのシームレスな統合により、信頼性が高くスケーラブルなデータベースソリューションを提供するために利用されています。

### **開発アプローチ**

- **フロントエンド開発:** ReactJSは、その効率性とユーザーフレンドリーさで知られており、ReactJSで開発されました。デザインはPCとモバイルデバイスの両方に対応しており、横向きと縦向きを含むさまざまな画面サイズや向きにシームレスに適応します。
- **バックエンド開発:** バックエンドには、最新のPythonフレームワークであるFastAPIが選択されました。Pythonの高速な起動時間は、Lambda関数にとって特に有益であり、サーバーレスアーキテクチャとうまく連携します。

### **アプリケーションの機能**

- **ユーザーインターフェース:** 技術的な専門知識がほとんどないユーザー向けに、シンプルで直感的な操作性を重視しています。インターフェースはわかりやすく、ナビゲーションやインタラクションを容易にします。
- **アクセシビリティ:** アプリケーションは、インターネットにアクセスできるあらゆるデバイスから普遍的にアクセスできるため、ユーザーはいつでもどこからでも自分の物件を管理できます。
- **機能性:** アパート管理の特定のニーズに対応するためにカスタムメイドされています。このアプリケーションは、アパート所有者の要件に合わせて、効率的な物件管理に不可欠な機能を網羅しています。

### **今後の機能強化**

- **継続的な改善:** ユーザーからのフィードバックや技術の進歩に基づいて、アプリケーションを定期的に更新・強化していく予定です。
- **機能拡張:** 自動家賃集金、メンテナンスリクエスト追跡、重要な更新のためのリアルタイム通知などの新機能を導入する可能性があります。
- **レポート機能の強化:** 財務および運用上の物件 oversight のためのより包括的なレポートツールを開発しています。
- **ユーザーエクスペリエンスの最適化:** ユーザーインターフェースとインタラクションデザインの改善に継続的に取り組み、アプリケーションがユーザーフレンドリーであり続け、新しいデザイントレンドに沿うようにします。
